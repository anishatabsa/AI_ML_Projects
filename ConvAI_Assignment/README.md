# CodeGen-Style Decoder-Only Transformer — Submission

## Contents

| File | Description |
|------|-------------|
| `codegen_transformer.ipynb` | Main notebook — complete pipeline (Tasks 1-6) |
| `technical_report.md` | Short technical report (hyperparameters, limitations, hardware) |
| `tokenizer/tokenizer.json` | Saved BPE tokenizer configuration |
| `tokenizer/vocab.json` | Tokenizer vocabulary mapping |
| `tokenizer/tokenizer_config.json` | Tokenizer metadata (vocab size, special tokens, etc.) |
| `tokenizer/train_corpus.txt` | Cleaned training corpus used for tokenizer training |

## Generated by running the notebook:
- `best_model.pt` — Best model checkpoint (lowest val loss)
- `final_model.pt` — Final model weights
- `model_config.json` — Model architecture hyperparameters
- `training_config.json` — Training hyperparameters
- `training_metrics.json` — Step-by-step loss log, val perplexities
- `generated_completions.json` — All 6 prompt completions with analysis
- `training_curves.png` — Training/validation loss curves
- `corpus_length_distribution.png` — Code length histogram
- `token_length_distribution.png` — BPE token length histogram
- `positional_encoding_visualization.png` — PE heatmap
- `lr_schedule.png` — Learning rate schedule visualization
- `generation_analysis.png` — Syntax validity, repetition, length charts

## Setup

```bash
pip install datasets tokenizers torch numpy matplotlib tqdm

```

## Task Mapping

| Task | Notebook Section | Description |
|------|-----------------|-------------|
| Task 1 | Data Preparation | Clean & deduplicate CodeSearchNet Python |
| Task 2 | Tokenization | Train BPE tokenizer from scratch |
| Task 3 | Dataset Engineering | CLM shifted input-label pairs |
| Task 4 | Input Representation | Token embeddings + sinusoidal PE |
| Task 5 | Architecture Design | Full decoder-only Transformer from scratch |
| Task 6 | Training & Evaluation | Training loop + 6 prompt generations + analysis |
